## ğŸ¯ Algorithm

The generator uses a **continuous path greedy algorithm**:

1. Place N pegs evenly around a circle (peg 0 at 12 o'clock)
2. Start at peg 0 with a white canvas
3. From the **current peg**, find the best unvisited connection that reduces the most "error"
4. Draw that string and **move to the new peg**
5. Repeat until no good connections remain or limits reached

This ensures the output is a **single continuous path** like:
